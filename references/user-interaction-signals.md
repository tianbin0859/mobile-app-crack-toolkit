# 用户交互信号速查表

> 来源：多次会话验证的用户行为模式
> 适用：所有与用户的交互场景，特别是破解/逆向/自动化任务

## 已验证信号

| 信号 | 含义 | 适用场景 | 首次验证日期 |
|------|------|---------|-------------|
| `全部` | 执行所有可用选项/功能，不需要逐个确认 | 多选项选择、批量操作、功能升级 | 2026-05-30 |
| `同意` | 立即继续执行，不需要详细解释或重复选项 | 批量操作、破解任务、APK分析 | 2026-06-09 |
| `继续` | 立即继续上一步操作，不需要再次确认或问 | 中断后恢复、长任务继续 | 2026-05-06 |
| `自行开始` | 自动执行不等待确认 | 预研、分析、编码任务 | 2026-05-22 |
| `你来清理` | 期望直接执行而非仅诊断 | 系统清理、故障修复 | 2026-05-06 |
| `选择最优方式` | 由Agent选择最佳方案并直接执行 | 技术选型、部署方案 | 2026-05-22 |
| `不用了/放弃/取消/没事` | 用户受挫，不要推动，可能稍后返回 | 任何任务中断 | 2026-05-06 |
| `暂时` | 暂停当前工作流，保留进度，不是放弃 | 长任务中断 | 2026-06-13 |
| `不用X` (如"不用7") | 跳过步骤X，不做该特定步骤 | 多步骤流程中的选择性跳过 | 2026-06-14 |
| `回退` | 撤销/回退到之前的版本/状态 | Git回退、版本回退 | 2026-06-17 |
| `上传` | 推送代码到Gitee/GitHub，不需要详细解释 | 项目完成后的仓库推送 | 2026-06-18 |

## 数字选择模式

- 用户从选项列表中选择时，**总是直接回复数字**（1/2/3）而非文字
- 不需要额外确认，数字即指令
- 已在多次交互中验证

## 响应策略

| 信号类型 | 响应方式 |
|---------|---------|
| 执行类（全部/同意/继续/自行开始） | 立即执行，不等待确认 |
| 跳过类（不用X） | 跳过指定步骤，继续后续流程 |
| 暂停类（暂时） | 保留进度，询问后续计划 |
| 放弃类（不用了/放弃） | 完全停止，询问真实需求 |
| 回退类（回退） | 执行回退操作，确认目标版本 |

## 注意事项

- 用户极其不耐烦重复确认和问答
- 当用户连续发同一条消息多遍、发"？？？"、或连续发"好了吗"/"什么情况"时，意味着他已经很生气了，需要立即执行，不要再问确认
- 用户说"入侵程序" = 自动化漏洞验证/攻击模块，要立即执行不反复确认
- 用户说"开发" = 直接执行不等待确认，"开发全自动" = 升级到更高级版本
