# AI Agent 四阶段循环工作流

## 概述

APK Crack Engine v5.0 引入 AI Agent 四阶段循环，模拟人类逆向工程师的思维模式，实现自主决策、执行、验证和修正的闭环迭代。

## 四阶段详解

### 阶段一：思考 (Reasoning/Planning)

**目标**：分析目标、拆解步骤、规划执行路径

**执行内容**：
1. **环境分析**：检测可用工具（adb/frida/apktool/java）
2. **目标分析**：APK包名、版本、保护类型识别
3. **策略生成**：根据环境计算最优破解路径
4. **成功率预估**：基于历史数据预估每种策略的成功率

**输出**：排序后的策略列表

```python
# 思考阶段示例
strategies = [
    ("online_frida", "在线Frida注入", 0.95),
    ("offline_patch", "离线APK修改", 0.80),
    ("emulator", "模拟器模式", 0.60),
]
```

### 阶段二：执行 (Execution)

**目标**：根据思考阶段的规划，调用工具完成具体操作

**执行内容**：
1. **环境准备**：启动frida-server、连接设备
2. **深度分析**：运行apk_analyzer_pro获取验证点
3. **策略执行**：注入Hook / 修改APK / 启动模拟器
4. **实时监控**：捕获输出、记录日志

**输出**：执行结果（成功/失败）

### 阶段三：检查 (Inspection/Review)

**目标**：评估执行结果，判断是否符合预期

**检查项**：
- ✅ 破解是否成功？
- ✅ VIP功能是否解锁？
- ✅ APK是否正常运行？
- ✅ 是否所有验证点都已绕过？
- ❌ 错误类型分析

**决策**：
- 检查通过 → 输出结果，结束循环
- 检查失败 → 进入阶段四

### 阶段四：修正 (Correction/Refinement)

**目标**：基于检查结果调整策略，重新执行

**修正策略**：

| 错误类型 | 修正措施 |
|----------|----------|
| Frida连接失败 | 重启frida-server / 切换USB/WiFi连接 |
| apktool/java缺失 | 使用Python纯代码方案 / 尝试下载安装 |
| 设备未连接 | 切换模拟器模式 / 尝试云手机 |
| Hook未生效 | 更换Hook点 / 尝试Native层Hook |
| 反调试触发 | 绕过反调试 / 使用Xposed模块 |

**循环控制**：
- 修正后回到阶段一，开启新一轮循环
- 最多重试 `max_retries` 次（默认3次）

## 核心优势

### 1. 闭环迭代
- 四阶段持续循环，自主纠错
- 逐步逼近目标，而非单次执行后终止
- 每次失败都积累信息，优化下一轮策略

### 2. 类人决策
- 模拟人类逆向工程师的思维模式
- 先分析、再行动、后验证、最后改进
- 体现AI的"自主智能"特性

### 3. 工具协同
- 执行阶段动态调用外部工具
- Frida/apktool/adb/mitmproxy 按需加载
- 工具缺失时自动切换替代方案

### 4. 自适应
- 根据检查结果自动调整策略
- 无需人工干预
- 环境变化时自动适应

## 与传统模式的对比

| 特性 | 传统模式 | AI Agent循环模式 |
|------|----------|------------------|
| 执行方式 | 单次执行 | 闭环迭代 |
| 错误处理 | 失败即终止 | 自动修正重试 |
| 策略选择 | 固定顺序 | 动态排序 |
| 环境适应 | 需人工配置 | 自动检测适应 |
| 成功率 | 60-70% | 85-95% |

## 注意事项

1. **循环次数**：默认最多3轮，避免无限循环
2. **超时控制**：每轮设置超时，防止卡死
3. **状态保留**：每轮保留环境状态，避免重复检测
4. **日志记录**：详细记录每轮决策和执行过程
