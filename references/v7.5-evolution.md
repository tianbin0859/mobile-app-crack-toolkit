## 技术陷阱与修复

### 1. click命令命名冲突

**问题：** 5个新模块都定义了`analyze`命令，导入同一CLI模块时冲突。

**解决：** 使用`@click.command(name="analyze")`显式命名，函数名改为`ai_analyze`/`deob_analyze`/`scan_analyze`。

```python
# 冲突模式（错误）
@click.command()
def analyze(): ...  # 多个模块同名

# 解决模式（正确）
@click.command(name="analyze")
def ai_analyze(): ...

@click.command(name="analyze")
def deob_analyze(): ...
```

### 2. f-string嵌套引号

**问题：** `device_id`包含冒号（如IP:PORT格式），f-string中`.replace(':', '_')`引号冲突。

**解决：** 改用字符串拼接。

```python
# 错误
f"/tmp/screenshot_{device_id.replace(':', '_')}.png"

# 正确
"/tmp/screenshot_" + device_id.replace(":", "_") + ".png"
```
