# 阿里云唯一方案架构决策

## 决策背景

用户明确选择**阿里云ECS作为唯一部署方案**，去除所有其他方式：
- ❌ ngrok（临时方案，域名不稳定）
- ❌ 花生壳DDNS（下载失败，不可靠）
- ❌ frp自建（复杂度太高）
- ❌ 本地部署（仅用于测试）

## 架构原则

### 1. 唯一部署方式

| 方式 | 状态 | 原因 |
|------|------|------|
| **阿里云ECS** | ✅ **唯一推荐** | 固定IP、国内稳定、完全可控 |
| ngrok | ❌ 禁止 | 域名随机变化，不适合生产 |
| 花生壳 | ❌ 禁止 | 下载链接失效，服务不可靠 |
| frp | ❌ 禁止 | 需要额外服务器，复杂度太高 |
| 本地部署 | ❌ 禁止 | 仅用于开发测试 |

### 2. 部署流程

```
购买阿里云ECS → 配置安全组(8080) → 上传deploy_aliyun.sh → 执行部署 → 公网访问
```

### 3. 服务器配置

| 配置 | 推荐值 |
|------|--------|
| 实例 | ECS 共享型 n4 |
| CPU | 1核 |
| 内存 | 2GB |
| 带宽 | 1Mbps |
| 系统 | CentOS 7.9 / Ubuntu 20.04 |
| 费用 | 约￥100/年（新用户优惠） |

### 4. 安全组规则

```
入方向：
- TCP 22    (SSH)
- TCP 8080  (授权服务API)
- TCP 80    (Web面板，可选)
```

## 管理命令

```bash
# 查看状态
systemctl status w528-auth

# 查看日志
journalctl -u w528-auth -f

# 生成卡密
cd /opt/w528-auth
python3 w528_auth_server.py generate --prefix ZL --days 30

# 重启服务
systemctl restart w528-auth
```

## Web管理面板

```
http://你的阿里云IP:8080/panel
```

功能：
- 卡密生成/列表/启用禁用
- 设备绑定查看/解绑
- 实时统计（总卡密/活跃设备/今日验证）
- 暗色主题（黑底+荧光绿）

## 卡密分级

| 前缀 | 功能 | 价格 |
|------|------|------|
| CL | 基础功能 | 低 |
| ZL | 高级功能 | 中 |
| GL | 全部功能 | 高 |

## 设备绑定机制

- 一机一码：卡密绑定设备指纹
- 最大设备数：可配置（1-10台）
- 心跳检测：每30分钟验证一次
- 离线缓冲：24小时离线可用

## 故障排查

| 问题 | 解决 |
|------|------|
| 无法访问8080 | 检查安全组规则 |
| 服务启动失败 | 检查Python3和依赖安装 |
| 卡密验证失败 | 检查卡密是否启用 |
| 设备绑定失败 | 检查是否已达最大设备数 |
